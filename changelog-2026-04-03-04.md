# Changelog — 2026-04-03 / 2026-04-04

## General summary

Full day of debugging and network reconfiguration - from missing internet access on VMs to migrating routers to a new IP range.

---

## 1. Fix internet on cluster VMs (10.13.13.100 - router-C)

**Problem:** VM `10.13.13.100` had no internet access.

**Cause:** The SNAT rule was missing on Proxmox for subnet `10.13.13.0/24`.

**Fix on Proxmox (10.90.90.13):**
```bash
iptables -t nat -A POSTROUTING -s 10.13.13.0/24 -o vmbr0 -j SNAT --to-source 10.90.90.13
```

---

## 2. Fix internet on Kubernetes nodes (192.168.70.0/24)

**Problem:** Kubernetes nodes (192.168.70.20-23) had no internet.

**Cause:** The SNAT rule on Proxmox was NATing to `10.88.88.230` - an IP that no longer existed on any interface.

**Fix on Proxmox:**
```bash
iptables -t nat -D POSTROUTING -s 192.168.70.0/24 -o vmbr0 -j SNAT --to-source 10.88.88.230
iptables -t nat -A POSTROUTING -s 192.168.70.0/24 -o vmbr0 -j SNAT --to-source 10.90.90.13
```

## 3. Fix DNS on all Kubernetes nodes

**Problem:** `systemd-resolved` on all nodes was using broken upstream DNS (9.9.9.9 / 149.112.112.112 timeout).

**Fix on each node (m, w1, w2, w3):**
```bash
rm /etc/resolv.conf
echo -e "nameserver 8.8.8.8\nnameserver 1.1.1.1" > /etc/resolv.conf
```

**Fixed nodes:**
- `192.168.70.20` (m)
- `192.168.70.21` (w1)
- `192.168.70.22` (w2)
- `192.168.70.23` (w3)

**CoreDNS restart** after fixing the nodes (it used old resolv.conf on startup):
```bash
kubectl rollout restart deployment coredns -n kube-system
```

## 4. Fix ArgoCD - incomplete CiliumNetworkPolicy

**Problem:** `argocd-application-controller` could not connect to `argocd-repo-server:8081` - `connect: operation not permitted`.

**Cause:** `CiliumNetworkPolicy` `argocd-repo-server-github-egress` only had `egress` rules. When Cilium sees a CiliumNetworkPolicy on a pod, it creates implicit ingress isolation as well - Kubernetes NetworkPolicies were no longer enough.

**Fix:** Added explicit `ingress` rules to the CiliumNetworkPolicy:
```bash
kubectl apply -f argocd/argocd-repo-server-github-egress.yaml
```

File: `argocd/argocd-repo-server-github-egress.yaml`

## 6. Migrate router-C (VM 104) to the new IP range

**Before:**
- IP: `10.13.13.100/24`, GW: `10.13.13.1`
- Bridge: `vnet1`
- VIP keepalived: `10.13.13.101`
- BGP router-id: `10.13.13.100`

**After:**
- IP: `10.90.90.9/24`, GW: `10.90.90.1`
- Bridge: `vmbr0`
- VIP keepalived: `10.90.90.101`
- BGP router-id: `10.90.90.9`

**Proxmox commands:**
```bash
qm set 104 --ipconfig0 ip=10.90.90.9/24,gw=10.90.90.1
qm set 104 --net0 virtio=BC:24:11:80:63:1D,bridge=vmbr0
qm reboot 104
```

**Keepalived - priority raised to 150 so it becomes MASTER:**
```bash
sed -i 's/priority 110/priority 150/' /etc/keepalived/keepalived.conf
sed -i 's|10.13.13.101/24|10.90.90.101/24|g' /etc/keepalived/keepalived.conf
# Added global_defs with script_security on
systemctl restart keepalived
```

**BGP:**
```bash
sed -i 's/bgp router-id 10.13.13.100/bgp router-id 10.90.90.9/' /etc/frr/frr.conf
systemctl reload frr
```

**iptables - updated everything to the new IP and cleaned rules:**
```bash
sed -i 's|10.13.13.101|10.90.90.101|g' /etc/iptables/rules.v4
iptables-restore < /etc/iptables/rules.v4
```
