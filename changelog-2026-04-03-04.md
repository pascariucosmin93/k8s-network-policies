# Changelog — 2026-04-03 / 2026-04-04

## Rezumat general

Zi completa de debugging si reconfigurare retea — de la lipsa internet pe VM-uri pana la migrarea routerelor pe o noua clasa de IP-uri.

---

## 1. Fix internet pe VM-urile din cluster (10.13.13.100 — router-C)

**Problema:** VM-ul `10.13.13.100` nu avea acces la internet.

**Cauza:** Lipsea regula SNAT pe Proxmox pentru subnet-ul `10.13.13.0/24`.

**Fix pe Proxmox (10.90.90.13):**
```bash
iptables -t nat -A POSTROUTING -s 10.13.13.0/24 -o vmbr0 -j SNAT --to-source 10.90.90.13
```

---

## 2. Fix internet pe nodurile Kubernetes (192.168.70.0/24)

**Problema:** Nodurile Kubernetes (192.168.70.20-23) nu aveau internet.

**Cauza:** Regula SNAT pe Proxmox facea NAT spre `10.88.88.230` — IP care nu mai exista pe nicio interfata.

**Fix pe Proxmox:**
```bash
iptables -t nat -D POSTROUTING -s 192.168.70.0/24 -o vmbr0 -j SNAT --to-source 10.88.88.230
iptables -t nat -A POSTROUTING -s 192.168.70.0/24 -o vmbr0 -j SNAT --to-source 10.90.90.13
```

---

## 3. Fix DNS pe toate nodurile Kubernetes

**Problema:** `systemd-resolved` pe toate nodurile folosea upstream DNS broken (9.9.9.9 / 149.112.112.112 timeout).

**Fix pe fiecare nod (m, w1, w2, w3):**
```bash
rm /etc/resolv.conf
echo -e "nameserver 8.8.8.8\nnameserver 1.1.1.1" > /etc/resolv.conf
```

**Noduri fixate:**
- `192.168.70.20` (m)
- `192.168.70.21` (w1)
- `192.168.70.22` (w2)
- `192.168.70.23` (w3)

**CoreDNS restart** dupa fixarea nodurilor (folosea resolv.conf vechi la pornire):
```bash
kubectl rollout restart deployment coredns -n kube-system
```

## 4. Fix ArgoCD — CiliumNetworkPolicy incompleta

**Problema:** `argocd-application-controller` nu putea conecta la `argocd-repo-server:8081` — `connect: operation not permitted`.

**Cauza:** `CiliumNetworkPolicy` `argocd-repo-server-github-egress` avea doar reguli de `egress`. Cand Cilium vede un CiliumNetworkPolicy pe un pod, creeaza izolare implicita si pe ingress — Kubernetes NetworkPolicies nu mai erau suficiente.

**Fix:** Adaugat reguli explicite de `ingress` in CiliumNetworkPolicy:
```bash
kubectl apply -f argocd/argocd-repo-server-github-egress.yaml
```

Fisier: `argocd/argocd-repo-server-github-egress.yaml`

## 6. Migrare router-C (VM 104) pe noua clasa de IP-uri

**Inainte:**
- IP: `10.13.13.100/24`, GW: `10.13.13.1`
- Bridge: `vnet1`
- VIP keepalived: `10.13.13.101`
- BGP router-id: `10.13.13.100`

**Dupa:**
- IP: `10.90.90.9/24`, GW: `10.90.90.1`
- Bridge: `vmbr0`
- VIP keepalived: `10.90.90.101`
- BGP router-id: `10.90.90.9`

**Comenzi Proxmox:**
```bash
qm set 104 --ipconfig0 ip=10.90.90.9/24,gw=10.90.90.1
qm set 104 --net0 virtio=BC:24:11:80:63:1D,bridge=vmbr0
qm reboot 104
```

**Keepalived — prioritate ridicata la 150 ca sa fie MASTER:**
```bash
sed -i 's/priority 110/priority 150/' /etc/keepalived/keepalived.conf
sed -i 's|10.13.13.101/24|10.90.90.101/24|g' /etc/keepalived/keepalived.conf
# Adaugat global_defs cu script_security on
systemctl restart keepalived
```

**BGP:**
```bash
sed -i 's/bgp router-id 10.13.13.100/bgp router-id 10.90.90.9/' /etc/frr/frr.conf
systemctl reload frr
```

**iptables — actualizat tot la noul IP si curatate regulile:**
```bash
sed -i 's|10.13.13.101|10.90.90.101|g' /etc/iptables/rules.v4
iptables-restore < /etc/iptables/rules.v4
```
